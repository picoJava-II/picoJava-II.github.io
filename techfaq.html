Overview of the picoJava-II Core

  1. Q: What is the picoJava-II core? 


    The picoJava-II core is a low-cost hardware engine, optimized to execute Java bytecodes in hardware. By accelerating the execution of these bytecodes in hardware, this core offers high performance without the need for Just-In-Time (JIT) compilers. The picoJava-II core has been designed specifically for cost-sensitive embedded applications, where performance cannot be compromised and JIT compilers, which require a larger memory footprint, are not a feasible solution. 

  2. Q: What applications is the picoJava-II core targeted for? 


    The picoJava-II core is targeted for "System-On-a-Chip" embedded applications, where Java performance is important and the use of JIT compilers, which require a lot of memory, is not feasible. Such applications (for example, set-top boxes, smart phones, and PDAs) usually are cost sensitive and use less memory. 

  3. Q: What instructions does the picoJava-II core support? 


    The core implements most of the Java virtual machine instructions directly while supporting a general-purpose instruction set capable of handling operating systems and code compiled from languages other than the Java language (such as C and C++). 

  4. Q: What is the Java performance of the picoJava-II core? 


    The picoJava-II core is optimized for Java performance, one measure of which is the SPECjvm98 benchmarks suite. In a system with 32 Mbytes of memory, we estimate the SPEC JVM98 best-run result for the picoJava-II core to be 4.2 at 120 MHz. See http://www.spec.org/osg/jvm98/results for SPEC JVM98 results on other processors. 

  5. Q: Does the picoJava-II core include a memory and I/O interface? 


    The picoJava-II technology, designed as a core rather than a complete microprocessor, does not include any memory or I/O interface logic. Instead, users can add their own logic for the memory or interface that they prefer. However, the core includes a Bus Interface Unit (BIU) with well-defined interface protocol to interface to the external world. Through a simple memory model, the BIU serves as a reference design for simulation of the core. 

    More About the picoJava-II Microarchitecture

  6. Q: What are the main functional units in the picoJava-II core? 


    The picoJava-II core consists of seven major units: 

    • Integer Unit (IU)
    • Floating Point Unit (FPU)
    • Instruction Cache Unit (ICU)
    • Data Cache Unit (DCU)
    • Stack Management Unit (SMU)
    • Bus Interface Unit (BIU)
    • Powerdown, Clock, Reset, and Scan Unit (PCSU)
  7. Q: What standard configurations of the design does the core support? 


    You can configure the size of the instruction and data caches in the Verilog from 0 (no cache) to 16 Kbytes (that is, 0 Kbyte, 1 Kbytes, 2 Kbytes, 4 Kbytes, 8 Kbytes, or 16 Kbytes). 

    Also, you can remove the Floating Point Unit (FPU) to save area in applications that do not require floating-point arithmetic. You must, however, provide floating-point emulation trap routines in software. 

  8. Q: Does the picoJava-II core implement the entire Java virtual machine? 


    The core is not the entire Java virtual machine in silicon. Rather, it is a microprocessor design optimized to run a small Java virtual machine implementation. As one step to achieve this goal, the core decodes all of the bytecodes defined by the Java virtual machine. Furthermore, it implements most of them directly in hardware. 

    The core can accelerate other aspects (for example, garbage collection) of a Java virtual machine with hardware. Bear in mind, however, that you must develop a significant amount of software to produce a compliant Java virtual machine that runs on the core. 

  9. Q: Does the picoJava-II core use any custom blocks that licensees have to design? If so, what are they? 


    The picoJava-II core uses a number of megacells that are to be designed as custom blocks. Examples are: the stack cache (a 64-entry register file with three read ports and two write ports); instruction cache data and tag RAMs; data cache data and tag RAMs; and the ROMs for microcode. These megacells are described as high-level behavioral models in the picoJava-II core Verilog design. 

  10. Q: What is the size and power dissipation for the picoJava-II core? 


    The picoJava-II core size depends on the following parameters: 

    • The size of the instruction and data caches
    • The process geometry and the number of metal layers used
    • The design methodology (ASIC versus the custom design approach)

    Licensees should synthesize the design with with their own library and compute the accurate gate count from the results. They can then estimate the size and power based on the gate count, frequency, megacell design data, and their process or design methodology. 

  11. Q: What is the frequency at which the picoJava-II core can run? 


    The picoJava-II core is a RTL design and can be targeted to any process and library. The frequency at which this core can run depends on the process and the library in which it is designed. 

    Designs with the picoJava-II Core

  12. Q: What are the deliverables for the picoJava-II core? 


    The picoJava-II design database includes the following: 

    • The Verilog RTL source for the core
    • The instruction simulator for the core
    • The simulation environment for the core
    • The source for the test suites that are used to verify the design
    • Documentation: 
      • picoJava-II Programmer's Reference Manual
      • picoJava-II Microarchitecture Guide
      • picoJava-II Verification Guide
      • Other documentation on the simulation and verification environment
  13. Q: What is the format in which the picoJava-II design is available? 


    The picoJava-II core, designed in Verilog's RTL language, consists of high-level behavioral megacell models and fully synthesizable logic for the rest of the core. You can synthesize the core with a cell library that you prefer. Synthesis results and the clock frequency that you achieve depend on your synthesis approach and cell library. 

  14. Q: Is the picoJava-II core designed for a particular process? 


    No. The Verilog model of the picoJava-II core contains no process-specific data, hence the core can be retargeted to any process. We recommend, however, that you choose a 0.25-micron or better process to achieve the expected clock frequency with minimum customization. 

  15. Q: What CAD design tools are used in the picoJava-II design? 


    Two major CAD tools are used in the design: 

    • Verilog: for RTL description
      The RTL is simulated or verified using Cadence Verilog-XL or Synopsys VCS.
    • Synthesis: the RTL is synthesized using the Synopsys Design Compiler
  16. Q: What platforms is the picoJava-II design environment based on? 


    The platform that is fully supported for the picoJava-II design environment is the SPARCTM/SolarisTM platform. Although you can port the simulation environment and tools to other platforms, they have not been tested or verified for those platforms. 

  17. Q: Can licensees define their own instructions? 


    One key factor in the compliance with the picoJava-II architecture is its Instruction Set Architecture (ISA). For tools and applications to function on all implementations of the picoJava-II technology, the ISA must be the same across all designs. Sun is considering provisions for reserving some picoJava-II bytecodes as "user-defined" bytecodes for specific applications and will provide details to licensees when they are available. Of course, you need comply with the ISA only if you intend to use the final design in commercial products that carry the appropriate picoJava-II technology logo. 

  18. Q: Can licensees make microarchitectural enhancements to the picoJava II core? 


    The picoJava-II core is an implementation of the ISA that is defined for this core. You can use this design as a reference to optimize, enhance, and reduce the costs of their implementations, or modify it for use in different applications. The final designs must, however, pass the compliance test suite if they are intended for use in commercial products that carry the appropriate logo. 

  19. Q: What is the picoJava-II Validation Test Suite? What does it test? 


    The picoJava-II Validation Test Suite (picoJava-II VTS), a suite of tests written mostly in the picoJava-II assembly language, is used to verify the functionality of the core. This test suite consists of many tests that are targeted at testing different units within the picoJava-II core and verifying the entire instruction set and corner cases for different instruction sequences. 

  20. Q: How do licensees access the picoJava-II VTS? 


    Licensees can access the picoJava-II VTS as part of the design database available on the Web. They can contact the Sun picoJava-II support group for technical support and assistance. 

  21. Q: Are chips that are based on the picoJava-II core available today? 


    The picoJava-II core is currently used in the Sun microJavaTM (MJ-701) test chip for software development, porting the JavaOSTM to the picoJava-II core, and demonstrating the performance of picoJava-II core against competitive architectures. 

    Some licensees will also have available test chips based on the picoJava-II core in the future. 

  22. Q: Has Sun designed any demo boards based on the picoJava-II core? 


    Sun has designed a demo board based on the microJava (MJ-701) chip, which is used for software development, porting the JavaOS to picoJava-II core, and performance evaluation of the picoJava-II core against competitive architectures. 

    Software and Development Tools for the picoJava-II Core

  23. Q: Is there a Java virtual machine for the picoJava-II core? 


    Sun is developing a Java virtual machine optimized for the picoJava-II core and may make it available to licensees in the future. 

  24. Q: What development tools are available for the picoJava-II core? 


    Sun is working closely with Metaware on the development of C and C++ compiler and other development tools for the picoJava-II core. Alpha versions of these tools are available to licensees and are being used for software programming. 

  25. Q: How can licensees access picoJava-II development tools? 


    Licensees can contact Metaware at http://www.metaware.com/spb_java.html for access to the development tools for the picoJava-II core.
